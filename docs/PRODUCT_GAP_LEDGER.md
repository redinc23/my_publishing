## 2. Evidence contract

This ledger is planned and closed against the **exact evidence vocabulary of
`docs/NEXT_GO.md` §1**:

| Status | Meaning | Satisfies a hard gate? |
| --- | --- | --- |
| NOT STARTED | No execution evidence exists | No |
| IN PROGRESS | Actions started; acceptance incomplete | No |
| BLOCKED | Dependency/access/defect/decision prevents completion | No |
| FAILED | Observed result violates pass criteria | No |
| PASSED | Exact-SHA criteria met and evidence reviewed | Yes, if current and complete |
| SUPERSEDED | Historical result retained but replaced | No |
| WAIVED | Approved exception, owner + residual risk recorded | **Never for an unchanged hard gate** |

**A waiver never makes an unchanged hard gate true.** A WAIVED item records an
owner-approved exception with residual risk; it does not flip G1–G13.

**Ledger `status` is planning state, SEPARATE from that evidence vocabulary.**
The YAML `status` field (`UNMAPPED | DUPLICATE | ALREADY_EVIDENCED | READY |
BLOCKED | SCOPE_DECISION | DEFERRED`) describes where an item sits in the
delivery plan. It never asserts gate evidence. An item only becomes **PASSED**
per the definition of done in `docs/AGENT_EXECUTION_PACKET.md` §6: exact-SHA
evidence with deployment ID, environment, UTC timestamp, actor, test command
and result, expected vs. observed, sanitized artifacts, and gate impact
recorded. Implementation alone is not done; a green test alone is not done.
